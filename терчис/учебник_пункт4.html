Пункт 4. Алгоритм Евклида

§1. Основные понятия и теоремы — Теория чисел

📖 Теория

Алгоритм Евклида — способ нахождения НОД двух чисел, основанный на свойстве 3 пункта 2: если $a = bq + c$, то $(a, b) = (b, c)$.

Алгоритм:
1. Ввести $a$, $b$ ($a \ge b \ge 0$).
2. Если $b = 0$, то Ответ: $a$. Конец.
3. $r :=$ остаток от деления $a$ на $b$; $a := b$; $b := r$.
4. Перейти на шаг 2.

В виде цепочки равенств:

$$a = bq_1 + r_1, \quad 0 \le r_1 < b$$ $$b = r_1 q_2 + r_2, \quad 0 \le r_2 < r_1$$ $$r_1 = r_2 q_3 + r_3, \quad 0 \le r_3 < r_2$$ $$\ldots$$ $$r_{n-2} = r_{n-1} q_n + r_n, \quad 0 \le r_n < r_{n-1}$$ $$r_{n-1} = r_n q_{n+1} \;\;\Rightarrow\;\; (a, b) = r_n$$
Линейное представление НОД. Из цепочки равенств алгоритма, поднимаясь снизу вверх, можно выразить $(a, b) = au + bv$ для некоторых целых $u$, $v$.
Пример. Найдём $(525, 231)$:
$$525 = 231 \cdot 2 + 63$$ $$231 = 63 \cdot 3 + 42$$ $$63 = 42 \cdot 1 + 21$$ $$42 = 21 \cdot 2 + 0 \;\;\Rightarrow\;\; (525, 231) = 21$$
Линейное представление:
$$21 = 63 - 42 \cdot 1 = 63 - (231 - 63 \cdot 3) = 525 \cdot 4 - 231 \cdot 9 \;\;\Rightarrow\;\; u = 4,\; v = -9.$$
